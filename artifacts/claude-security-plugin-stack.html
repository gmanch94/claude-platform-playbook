Claude Security plugin — where it fits

Six layers of code-security coverage on the Claude platform. The plugin announced in beta is one of them: the on-demand deep scan.

One command, three jobs

Scan the codebase

Whole repo or a focused area. The plugin reads the repo first and shows file count and relative cost per option before anything runs.

Works with or without version control. Reads the working tree.

Scan a set of changes

Branch diff vs. base, an open PR's diff, or one commit — scan commit abc1234.

Committed changes only. Commit or stash WIP first. Git only.

Suggest patches

Pick findings by ID. Each patch is drafted in a scratch copy of the repo — source files stay untouched until you apply one.

Stale findings are skipped, not patched blind. Git only.

The gate — check before promising anyone a scan

Hard requirements

  • Claude Code v2.1.154+, paid plan
  • Dynamic workflows on (Pro: enable in /config)
  • python3 3.9.6+ on PATH — stdlib only, nothing installed
  • Git, for change scans and patching
  • Linux, macOS, or Windows

Install

If the marketplace isn't found:

/plugin marketplace add anthropics/claude-plugins-official

Then install claude-security and activate with /reload-plugins. Runs best in auto mode, so agents aren't stopped at every step.

What lands in your repo

  • CLAUDE-SECURITY-RESULTS.md — ID, impact, exploit scenario, severity, confidence, fix
  • CLAUDE-SECURITY-RESULTS.jsonl — machine-readable
  • CLAUDE-SECURITY-REVISION-<commit>.json — ties the report to the code it describes

One timestamped directory, carrying its own .gitignore. That's the only change a scan makes.

Name the failure modes

Cost is real and un-fixed

"May take a while, may use a significant number of tokens" is Anthropic's own wording. Multi-agent plus adversarial review means many calls. Budget it as a campaign, not a command.

Session-bound

Claude Code must stay open for the run to finish. This is a workstation tool. Unattended, monitored scanning is layer 5 or your CI — not this.

The report is now a sensitive asset

It is a list of live, exploitable weaknesses with exploit scenarios attached. The default .gitignore protects you. Deleting it "for the audit trail" is a deliberate decision, not a formality.

Don't quote the wrong number

The 30–40% drop in security comments on PRs belongs to the guidance plugin (layer 1), not this one. No published precision/recall for this plugin yet — it's beta.
