ROI / Value Worksheet As of 2026-06

The value side of the decision. cost-calculator.html sizes what you pay; this sizes what you get — and nets the two into payback and a value-to-cost ratio. Built to be defensible to a CFO, not to flatter the project: conservative defaults, an explicit realized-capture discount, and the soft-value trap called out. Garbage baseline in, garbage ROI out.

1. The workload

Headcount whose time the workload touches.
Time spent on the part Claude could assist — not the whole job.
Fully-loaded (salary + benefits + overhead), not base pay. Default ≈ $75.

2. The honest discounts

Fraction of the task Claude actually touches. Capped low on purpose — whole-job automation is rare.
The anti-theater dial. Time "saved" only counts if it converts to more output or fewer hires. Most orgs realize 40–60% of the theoretical saving in year one. Leave low until you have evidence.
Value does not start in month 1. Linear ramp assumed over this window.

3. Costs + hard value

Steady-state, from cost-calculator.html — not the pilot month. Include seats + API + a build/maintenance line.
Only measurable non-time value: avoided penalty, recovered revenue, deflected vendor cost. Leave 0 if you'd have to hand-wave it.

How this is computed

Gross time value = people × hours/wk × 52 × assist% × capture% × loaded rate.

Net = gross + other hard value − annual Claude spend. Ratio = (gross + hard) ÷ spend. Payback = spend ÷ steady-state monthly net (ignores ramp; the ramp-adjusted first-year figure is shown separately).

The sensitivity table re-runs the net at capture −20 / as-set / +20 points. If the case only works at the optimistic row, it doesn't work. Order-of-magnitude estimate, not a commitment — the inputs are yours to defend.

Five ways this number lies (read before you present it)

  • Double-count. The same hour saved cannot appear in two use cases' ROI. Maintain one value ledger across the portfolio; sum nets there, not in separate worksheets.
  • No baseline. If you can't measure today's cycle time / cost-per-task, the "before" is a guess and the ROI is fiction. Measure the current state for two weeks first — that's a Week 0 task in adoption-playbook.md.
  • Productivity theater. 30 minutes saved per person per day is not value until it becomes shipped output or an avoided hire. The realized-capture dial is where you discount it — keep it honest.
  • Soft value dominating. If "other hard value" is carrying the case, the case is weak. A banner fires when it exceeds the time value. Strip it back to what you'd defend in an audit.
  • Pilot-cost optimism. The pilot month understates steady-state spend (low volume, no maintenance, no second team). Pull the spend figure from cost-calculator's steady-state projection, then add a maintenance line.

What this worksheet deliberately does not count: brand/morale halo, "future optionality," and time saved that you can't tie to output. Leaving them out is the point — an ROI a finance lead can't poke holes in beats a bigger one they can.
