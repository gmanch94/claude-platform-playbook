The workflows

1 · Security guidance plugin — in session

/plugin install security-guidance@claude-plugins-official — three checkpoints at three depths, wired as hooks.

GAall Claude Code usersfree catches the most volume never blocks a write
Guidance is not enforcement. This layer surfaces findings for Claude to fix. It does not block writes and does not guarantee every violation is caught — a guidance rule saying to ignore a vulnerability class does not suppress those findings. For a hard stop, pair it with a blocking hook or a CI check.

2 · /security-review — on demand, single pass

Built in. No install, no plan gate, no marketplace.

GAall plansincluded usage cheapest pre-PR check nothing durable comes out

3 · Claude Security plugin — on demand, deep scan

/claude-security — one command, three jobs. The newest layer, in beta.

BETApaid plan · CC v2.1.154+ python3 3.9.6+ · git may use significant tokens session must stay open

4 · Code Review and /code-review ultra — on pull request

The only layer that runs whether the author remembered or not — and the one with a real per-review price.

research previewTeam, Enterprise ~$15–25 per reviewusage credits, not plan usage unavailable under ZDR ultra: no Bedrock, GCP, Foundry
The governance read. Two of the strongest layers are off the table under Zero Data Retention, and ultrareview is also off the table on Bedrock, Google Cloud's Agent Platform, and Microsoft Foundry. If that is your posture, your realistic stack is layers 1, 2, 3 and 6 — decide that at design time, not at rollout.

5 · Claude Security (managed product) — hosted, always on

The only layer that scans without a developer in a session.

Adjacent — different axis, often confused

Runtime containment (Claude Code)

Sandboxed Bash (/sandbox) for filesystem and network isolation, the working-directory write boundary, denyRead rules, Accept Edits mode.

Constrains what the agent can do, not what the code contains. Full treatment in agentic-threat-model.md and claude-code-enterprise-config.md.

Usage governance (Compliance API)

Programmatic access to Claude Enterprise conversation content and activity events, wired into 60+ DLP, SIEM, eDiscovery and identity platforms.

Governs how people use Claude, and scans no code at all. Full treatment in usage-compliance-monitoring.md and legal-hold-ediscovery-runbook.md.

Your own CI scanners

Keep them. The layers above are additive to SAST and SCA, not a migration path off them.

Claude reasons about your code; it does not replace language rule packs, dependency scanning, or policy enforcement. A plan that retires CI because "Claude reviews it now" is itself a finding.
