Claude Platform Playbook

Enterprise Data Boundaries As of 2026-07

Per-feature trust-zone diagrams for a Claude Enterprise deployment — where data crosses, where it's processed, where it's stored, and what the BAA covers. For CISOs, security architects, and compliance leads. Every boundary claim traces to governance-overlay.md (the source of truth); verify at privacy.claude.com.
The prompt is the trust boundary. Anything you put in a prompt (or a connected tool result, or an uploaded file) is on Anthropic's infrastructure for the duration of the request — and longer if you use Files API, Memory, or caching. Everything left in your VPC, databases, and source systems stays yours. These diagrams show, per feature, exactly what crosses that line. Architect tool layers to retrieve only what the model must see. [governance-overlay.md §1]

Cross-cutting controls — read these before the per-feature cells

  • No-train ≠ no-store. Commercial default is no-training-on-your-data; it does not mean data isn't stored. Default at-rest retention is 30 days (§11). [§2]
  • ZDR is not default, and not on the Enterprise product UI. Zero Data Retention covers the Messages API + Claude Code only (enterprise agreement, Anthropic approval) — never chat / projects / artifacts. Trust & Safety classifier scores are retained up to 7 years even under ZDR. [§3]
  • inference_geo is first-party-API + Claude Platform on AWS only (Opus 4.6+ and Sonnet 4.6+, confirmed — Sonnet 5 confirmed 2026-07-06: platform pricing doc explicitly covers "Sonnet 4.6 and later models"; 1.1× multiplier when inference_geo: "us"). It does not apply to the product UI, to Managed Agents, or to Bedrock/Vertex/Azure AI Foundry (those use the hyperscaler region or deployment type). Workspace geo governs at-rest storage + processing (code exec, image transcoding). [§5]
  • The BAA is not automatic and is per-surface. An Enterprise admin must activate HIPAA compliance + sign. Covered features differ between the first-party API and the Enterprise product surface — the same capability (code execution) is covered on one and not the other. The BAA does not extend to your MCP servers. [§4]

Where it bites: two identity modes hide under one name. Channel tagging runs under your org's Claude identity with admin-scoped tools/data/repos, billed to the org, and worked by whoever tags it — the confused-deputy risk is sharper here than on Cowork or Claude Code because multiple people share one agent identity (see agentic-threat-model T8). Slack's own copy of the conversation follows your org's Slack retention policy — a second store Anthropic doesn't control. BAA/ZDR/inference_geo are not stated anywhere in Anthropic's Claude Tag materials — treat as not covered until confirmed, the same posture as Cowork and Claude Design, not a silent third "probably fine."

A stolen or compromised phone with an active claude.ai session is now functionally a stolen credential to that machine — same shape as agentic-threat-model T6, just with a personal device as the new attack surface.

Where it bites: the local-data story is genuinely strong — raw datasets and compute stay on your machine, code runs in a sandbox, and you approve each folder, network host, and remote job before the agent touches it (all sourced). What crosses is the prompt and the model's response, held under standard retention — not a special persistent store like Design (#17) or Files API (#6). The trap is governance, not data locality: Claude Science ships on consumer Pro/Max plans with no BAA, its BAA, ZDR, and inference_geo residency are unpublished ("contact sales for specific needs"), and it's aimed at exactly the genomics / clinical / proteomics data most likely to be regulated. Anthropic's own docs state it is not intended for clinical or diagnostic use. Its 60+ connectors also reach external scientific databases — each host is approval-gated, but that's a second egress path to scope. Treat governance as not stated → not covered until Anthropic confirms it, the same posture as Cowork (#16) / Design (#17) / Tag (#18) — don't read "raw data stays local" as "regulated data is safe here."
