⚠ The trap the diagram makes visible

The Data-Science pipeline authenticates on a data-scientist's setup-token — a person-bound credential — and hits the same inference path as a proper workspace key. It works, so nobody notices.

Then: the scientist offboards → the pipeline breaks; the audit log shows their name, not the workload; a token leak is a year of inference. Fix: give it a WIF svac_ (like the CI runner) or a workspace key. Guide §4.

The hyperscaler attribution gap

The Bedrock workload uses an AWS IAM role — a real machine identity, but your cloud's. It sits outside the Anthropic workspace: the identity and its logs live in AWS.

Don't expect Anthropic's Console to show it. Attribute and audit on the cloud-IAM side. Guide §3 · §6.

Least privilege, one identity per workload

Keyless WIF mints the CI runner a short-lived token to create clinical-ai-prod (Admin) and run inference (workspace) — no long-lived secret in the runner.

Each workspace is its own blast-radius boundary. Never a shared key across workloads (blended audit trail), and never the Admin key for inference (org-wide blast radius). Guide §5.
