How recommendations are scored

Each option has a fit profile across the 5 axes. The worksheet computes a fit score per option = how closely your inputs match each profile's strengths. TCO bands are order-of-magnitude annual ranges, not commitments.

No option is universally best. The dominant signal is usually regulatory + scale; customization depth flips Skills/MCP from optional to essential.

Decision frames (cheat sheet)

  • Regulated workload + need latest models → Claude direct. Hyperscaler routes lag on model versions.
  • Regulated workload + already deep on AWS / GCP → Claude via Bedrock or Vertex. Procurement path matters more than version lag.
  • Generic chat / RAG, no proprietary edge → Packaged SaaS likely cheaper unless volume is high.
  • High-volume, low-margin, latency-tolerant batch → Run the math both ways. Claude Haiku 4.5 + batch is hard to beat. Open-source self-hosted only when volume + ops capacity are both real.
  • Deep proprietary domain → Claude direct + Skills + MCP + plugins. The customization stack is what makes "build" win over "buy SaaS."
  • No internal AI/ML expertise → Packaged SaaS or Claude direct with a small build team. Skip open-source self-hosted entirely.
  • Multiple teams adopting → Claude direct + plugins distribution. Standardize the platform; let teams build on it.
